# 00 · 项目总览

## 项目定位

Android Builder 是一个运行在安卓手机上的自动开发 Agent 控制 App。用户创建项目并输入需求后，App 调用 OpenAI-compatible 模型生成工程计划、拆分执行任务、逐步写入 Android 项目源码，再在手机本地构建 APK，并通过系统安装流程安装。

这个仓库本身是“控制 App”，它生成和构建的用户项目存放在 App 私有目录中，不在仓库源码目录里。

## 当前核心能力

- 多项目管理：新建、搜索、重命名、删除项目。
- 本地持久化：项目、对话、工程计划、执行任务、构建任务、APK 产物保存在本机。
- 模型接入：支持 OpenAI、DeepSeek、MiniMax、自定义 OpenAI-compatible endpoint。
- 线性项目页：执行任务、对话/需求记录、构建日志统一显示在一个时间线列表中，并对冗余状态消息做降噪。
- 文件侧滑栏：项目源码在项目页侧滑栏中浏览和预览。
- 计划驱动开发：先生成工程计划，再拆分任务，再逐步执行文件操作。
- 本地 APK 构建：默认嵌入式 runtime，保留外部 Termux 兼容模式。
- 手动修复：构建失败后由用户点击 Repair 按钮触发修复；只有 `BuildFailureClassifier` 判定为可由模型修复的失败才提供该按钮，环境/网络/权限类失败不提供。修复完成后不自动重新构建，由用户再次点击 Build。
- APK 安装：通过 Android PackageInstaller 安装最近一次产出 APK 的构建，仅在系统回调 `STATUS_SUCCESS` 时判定安装成功。
- 多语言：跟随系统、English、中文。

## 技术栈

- 主语言：Java。
- UI：原生 Android View/XML、Material Components、AppCompat。
- 构建：Gradle Wrapper、Android Gradle Plugin 8.7.3。
- 模块：`:app` 主 App，`:embedded-runtime` 私有构建环境管理。
- 存储：SQLiteOpenHelper + App 私有文件目录。
- 模型协议：OpenAI Chat Completions 兼容接口。
- `minSdk`：24。
- `targetSdk`：28。原因是嵌入式 runtime 需要执行 App 私有目录内的 arm64 二进制，Android 10+ 对更高 targetSdk 有限制。

## 关键包职责

| 路径 | 职责 |
| --- | --- |
| `app/src/main/java/com/androidbuilder/ui/` | Activity、项目时间线、设置页、UI 状态策略 |
| `app/src/main/java/com/androidbuilder/agent/` | 模型调用、计划生成、任务拆分、文件操作解析/写入、源码策略防线 |
| `app/src/main/java/com/androidbuilder/backend/` | 构建后端抽象、嵌入式 runtime 构建、Termux 构建、Gradle 归一化 |
| `app/src/main/java/com/androidbuilder/data/` | SQLite schema 和仓储 |
| `app/src/main/java/com/androidbuilder/model/` | 数据记录对象和模型输出对象 |
| `app/src/main/java/com/androidbuilder/server/` | Termux 构建模式的本地 HTTP 回调服务 |
| `app/src/main/java/com/androidbuilder/termux/` | Termux RUN_COMMAND 调用和权限检测 |
| `app/src/main/java/com/androidbuilder/install/` | APK 安装 |
| `app/src/main/java/com/androidbuilder/util/` | 设置、文件工具、语言、本地工作保活 |
| `embedded-runtime/src/main/java/com/androidbuilder/embeddedruntime/` | App 私有 runtime 目录和 bootstrap 管理 |

| `AndroidSourceGuard` | 生成源码和资源引用校验 |

| `EmbeddedRuntime` | 维护 `files/runtime` 并检查工具链 |

## 生成源码约束

接手改 Agent 生成链路时，优先保持这些约束：

- 生成 App 使用 Java + XML，不生成 Kotlin 或 `.kt` 文件。
- 不使用 Compose、DataBinding、ViewBinding、Kotlin synthetic。
- 不使用 Java lambda 或 `->`，监听器使用匿名内部类。
- Java 中使用的 `R.id.*` 必须对应 XML `android:id`。
- 代码和 XML 引用的 string/color/style/layout/drawable/mipmap 必须存在。
- Fragment 中不能裸用 `findViewById()`，要从 root view 或 `requireView()` 取。
- Gradle 使用 `com.android.application`，AGP 8.7.3，Java 8 source/target。
- `AndroidManifest.xml` 不写 `package="..."`，包名由 Gradle `namespace`/`applicationId` 管理。

这些约束分布在：

- `OpenAiClient` 的 system prompt。
- `DependencyGuard`。
- `AndroidSourceGuard`。
- `AndroidGradleNormalizer`。

## 接手注意事项

- `app/src/main/assets/runtime/bootstrap-aarch64.zip` 可能超过 400 MB，不要直接提交到普通 GitHub 仓库，除非已配置 Git LFS 或确认小于 100 MB。
- `.idea/*` 多数是本机 IDE 状态，提交前确认是否真的需要。
- README 里可能有历史描述；当前代码生成目标是 Java + XML。
- 修改 DB schema 时递增 `DatabaseHelper.DB_VERSION` 并补 `onUpgrade()`。
- 修改模型 prompt 时同步考虑 parser、guard、测试，不能只改 prompt。
- 修改 `targetSdk` 前必须重新评估嵌入式 runtime 执行私有二进制的限制。
